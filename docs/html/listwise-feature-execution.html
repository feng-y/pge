Listwise 特征执行语义

Listwise 路径下特征执行的完整语义。process_listwise 进入后，先由 CalculateFeasignModule 串行执行 M 个 item 的 User+Item 特征，再由 CalculateSeqFeasignModule 并行执行 K 个 unique sequences 的 Seq 特征。最终 ListwiseQSModel::generate_input() 消费两次执行的输出，组装为 Galaxy input tensors。

listwise_qs_model.cpp :83–264

Galaxy Input 组装：generate_input() 消费两次特征执行的输出，按三种方式转换为模型 tensors：

User（:122）— 按 slot_idx_map 过滤，直接加入 inputs。Galaxy shape: [1, ...]，模型内部 broadcast。

Item（:168）— 按每条 sequence 的 item indices 从 M 个 item tensors 中 gather。Galaxy shape: [N × list_size, ...]。

Seq（:190）— 每个 batch row 找到其 unique sequence 的 index，create_padded_map 展开为 [N × list_size]，有效位置取 feasign，padding 位置填 -1。然后 rehash + build tensor。Galaxy shape: [N × list_size, ...]。

Masks（:264）— masks[i * list_size + j] = 1.0f if valid, 0.0 if padding。batch_tensor = N × list_size。

Agent Gotchas

  • Seq convert_to_tensor() 只产出 map，不构建 tensor。名称有误导性。
  • User 特征只从 idx == 0 的 item 执行上下文收集。
  • Seq pass 复用第一次执行准备好的 FeaContext（共享状态）。
  • ParallelSeqFeaRunner 第一个 seq 先单独执行（初始化 FS 静态变量），再并行剩余。
  • Listwise Item 执行是串行的（不走 ParallelFeaRunner），因为 M < 100。

  • SequenceScoreType打分意图载体，携带 _sequence（item indices 排列）
