# I002: Update Final Plan Gate For Combined Plan + Issue Files

Intent: make the validator accept the new split contract without weakening execution readiness.

## introduction

Update Final Plan Gate so it validates the combined `plan.md` stable contract, compact issue index, referenced issue files, and index/file consistency.

## evaluation

This is the validator foundation. It prevents producer mistakes from routing `READY_FOR_EXECUTE`, especially missing issue files, embedded issue bodies, and index/file disagreement.

## context

- Requirement goal served: validator side of the issue-file contract migration.

## task

Update Final Plan Gate and self-review references so `READY_FOR_EXECUTE` requires a valid compact issue index, existing referenced issue files, complete issue contracts, and index/file consistency.

## behavior_contract

- Current Behavior: validator wording assumes executable issue bodies can live inside `plan.md`.
- Desired Behavior: validator accepts only issue-file plan shape for executable plans and blocks embedded executable issue bodies.
- Behavior Delta: Final Plan Gate validates both stable plan fields and referenced issue files before execution is allowed.
- Key Interfaces: `skills/pge-plan/references/plan-gate.md`, `skills/pge-plan/references/final-plan-gate.md`, `skills/pge-plan/references/self-review.md`, `skills/pge-plan/evals/evals.json`.
- Trigger Predicate: plan route attempts to become `READY_FOR_EXECUTE`.
- Output Admission Predicate: gate can reject missing file, malformed issue file, embedded issue-body plan, dependency mismatch, and index/file status disagreement.
- Out Of Scope Confirmed: do not make Final Plan Gate a scheduler and do not create a new validator authority.
- What Not To Infer: do not accept a plan because issue bodies are present in `plan.md`; embedded bodies are an upgrade input, not executable shape.

## scope

### Do

- Add combined plan + issue-file validation requirements.
- Define embedded issue-body rejection in gate/self-review wording.
- Add or update eval scenarios for missing issue files and embedded issue-body rejection.

### Do Not

- Do not change `pge-exec` implementation wording beyond handoff expectations.
- Do not add compatibility handling for old embedded plans.
- Do not change final route vocabulary.

- Do not change: Final Plan Gate remains the only execution authorization validator.

## acceptance

- Final Plan Gate validates both `plan.md` and referenced issue files.
- Missing issue files or incomplete issue file fields block `READY_FOR_EXECUTE`.
- Embedded full issue bodies under `## issues` block `READY_FOR_EXECUTE` until `pge-plan` upgrades them into issue files.
- Gate checks that `plan.md ## issues` is schedulable from index fields without opening every issue file.
- Gate checks that dependencies, verification coupling, and execution type agree between the index and issue files.
- Route/status/verdict vocabulary remains unchanged.
