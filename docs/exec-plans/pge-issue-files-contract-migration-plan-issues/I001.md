# I001: Add Issue File Plan Contract To `pge-plan`

Intent: make `pge-plan` produce compact plan indexes plus separate issue files.

## introduction

Teach `pge-plan` to produce the new artifact shape: stable `plan.md` with a compact issue index, plus full issue contracts under `issues/Ixxx.md`.

## evaluation

This is the producer foundation. It is necessary but not sufficient on its own because new producer output must be validated by the Final Plan Gate in I002 before it is safe for execution.

## task

Update the `pge-plan` producer surface so new executable plans are authored as a stable `plan.md` with a compact `## issues` index plus separate `issues/Ixxx.md` files.

## behavior_contract

- Current Behavior: producer docs/templates allow or imply full executable issue bodies under `plan.md ## issues`.
- Desired Behavior: producer docs/templates require issue-file plans by default and describe `## issues` as an execution index.
- Behavior Delta: move full issue execution contract shape into `skills/pge-plan/templates/issue.md` and keep the plan template focused on stable plan fields and issue metadata.
- Key Interfaces: `skills/pge-plan/SKILL.md`, `skills/pge-plan/templates/plan.md`, `skills/pge-plan/templates/issue.md`, README files, Plan Engineering Review references.
- Trigger Predicate: any new PGE plan generation or fast-adopt materialization that routes `READY_FOR_EXECUTE`.
- Output Admission Predicate: produced plan has index rows with file paths and issue files with complete execution contracts.
- Out Of Scope Confirmed: do not change route/status/verdict vocabulary and do not introduce a second canonical plan artifact.
- What Not To Infer: do not treat issue files as final acceptance objects or as a GitHub/Jira issue tracker.

## scope

- Update producer instructions and templates for the issue-file artifact shape.
- Add or align the issue-file template.

### Do Not
