# I003: Update `pge-exec` To Require And Load Issue Files Lazily

Intent: make execution consume only issue-file plans and reject embedded issue-body plans before run creation.

## introduction

Update `pge-exec` to read the issue index first, reject embedded issue-body plans, lazy-load selected issue files, and build Generator briefs from the issue file plus `plan_context_packet`.

## evaluation

This is the consumer foundation and the highest protocol-drift risk. It must land after gate support so execution does not accept malformed split artifacts or continue treating embedded issue bodies as executable.

- Requirement goal served: consumer side of the issue-file contract migration.
- Upstream decisions: exec consumes only issue-file plans; embedded issue-body plans route to `pge-plan` for upgrade before run creation.

## task

Update `pge-exec` source loading, validation, scheduling, and Generator handoff wording so execution is index-first, issue-file-lazy-loaded, and protected by `plan_context_packet`.

## behavior_contract

- Current Behavior: exec wording can treat `plan.md ## issues` as the source of full issue execution packs.
- Desired Behavior: exec builds an issue graph from index fields, rejects embedded-body plans before run creation, loads selected issue files on demand, and constructs briefs from selected issue fields plus `plan_context_packet`.
- Behavior Delta: replace embedded issue extraction with issue-file shape detection, pre-run rejection, lazy loading, and run-scoped adaptation recording.
- Key Interfaces: `skills/pge-exec/SKILL.md`, `skills/pge-exec/handoffs/generator.md`, `skills/pge-exec/references/generator-rules.md`, `skills/pge-exec/references/evaluator-thresholds.md`.
- Trigger Predicate: `pge-exec` is invoked on a canonical task plan.
- Output Admission Predicate: no run is created for embedded, malformed, missing-file, or index/file-inconsistent plan shapes.
- Out Of Scope Confirmed: do not mutate `plan.md` or issue files during execution to approve scope changes.
- What Not To Infer: do not load all issue files by default and do not treat issue completion as final proof.

## scope

### Do

- Define issue-file plan shape as required execution input.
- Define `non_canonical_plan_shape` as an exec failure reason before run creation.
- Document issue graph construction from index fields.
- Document lazy loading of selected issue files and `plan_context_packet` brief construction.
- Keep adaptation notes run-scoped.

### Do Not

- Do not implement or document an embedded-plan fallback executor.
- Do not add task-level `exec/progress.md` or `exec/decision-log.md`.
- Do not turn `pge-plan` into a scheduler.

## acceptance

- Exec requires issue-file plan shape.
- Exec rejects embedded full issue bodies as non-canonical input and routes to `pge-plan` for upgrade.
- Exec validates issue file existence before dispatch.
- Exec reads `plan.md` issue index first.
- Exec reads selected `issues/Ixxx.md` on demand.
- Generator brief is derived from the selected issue file plus stable plan context.
- Exec builds a shared `plan_context_packet` once per run and attaches it to every issue brief.
- Exec builds an issue graph from index fields before loading full issue files.
- Exec schedules by dependency, target-area overlap, verification coupling, execution type, security risk, and current runtime truth.
- Exec records issue-local adaptation in `implementation-notes.md` instead of editing plan or issue files.
- Exec routes upstream when missing fields, malformed issue files, or index/file disagreement make scheduling unsafe.
- Run artifacts remain under `runs/<run_id>/`.
