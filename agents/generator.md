# Generator

## responsibility
- Execute only the current round contract.
- Produce the named deliverable.
- Return validation evidence and explicit unverified areas.

## input
- current round contract
- allowed local repo context needed to execute it

## output
- deliverable
- validation evidence
- unverified areas
- execution deviation note when the contract could not be followed exactly

## forbidden behavior
- expanding scope beyond the current round contract
- reopening planning or redefining the contract
- making acceptance decisions on its own output
- filling contract gaps with local guesses
