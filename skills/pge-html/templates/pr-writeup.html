Why this change

pge-handoff had accumulated two distinct responsibilities: session handoff (temporary) and knowledge extraction (permanent). This blurred boundaries and made handoff output unreliable for both use cases. The PR splits them into pge-handoff (observer-only, Matt Pocock style) and pge-knowledge (quality-evaluated, ce-compound inspired).

Before
  • pge-handoff contained save/extract/restore modes
  • Handoff tried to manage pipeline state
  • Knowledge extraction mixed into session handoff
  • No quality threshold for durable knowledge
After
  • pge-handoff: observer + session summary only
  • Handoff no longer touches pipeline state
  • pge-knowledge: separate skill with 8-dim quality scoring
  • Knowledge needs 14+ score before promotion to repo

What changed

skills/pge-handoff/SKILL.md Modified

Cut extract and restore modes. Now a pure observer that writes session summaries — nothing durable, nothing pipeline-aware. Reduced from 267 lines to a focused ~60-line handoff doc generator.

straightforward removal
skills/pge-knowledge/SKILL.md Added

New standalone skill. 8-dimension quality scoring (0–16 scale). Extract only promotes to repo when score ≥ 14. Refresh mode uses five outcomes (Keep / Update / Consolidate / Mark stale / Recommend deletion).

Quality gate: score ≥ 14 → promote 10–13 → requires supplemental evidence <10 → rejected; log as candidate only
new pattern needs careful review
CLAUDE.md Modified

Updated truth hierarchy and workflow authority sections to reference both pge-handoff and pge-knowledge. Added explicit role separation: handoff = temporary, knowledge = durable. Added pge-knowledge to first reads order.

Testing notes

  1. Run the contract validator — all pipeline skills should pass:
    ./bin/pge-validate-contracts.sh
  2. Verify handoff docs are lightweight (aim for <100 lines output):
    /pge-handoff save
  3. Test knowledge scoring on a real run artifact:
    /pge-knowledge score .pge/tasks-auth/runs/run-001/summary.md
  4. Check that CLAUDE.md loads both skills in order:
    head -20 CLAUDE.md
  5. Confirm handoff and knowledge outputs are decoupled — run both on the same session:
    /pge-handoff save && /pge-knowledge review
